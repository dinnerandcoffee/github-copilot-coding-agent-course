# Lab 01. TODO Basic  
Copilot과 함께 만드는 첫 실습 프로젝트

이 실습은 이 강의 전체에서 사용될 **기준 실습(Lab Template)**이다.  
Copilot Coding Agent를 사용해 **작은 프로젝트를 처음부터 끝까지** 진행한다.

---

## 실습 목표

이 Lab가 끝나면 다음을 경험하게 된다.

- Copilot에게 프로젝트를 설명한다
- 기능을 단계적으로 맡긴다
- 테스트 → 수정 → 검증 루프를 만든다
- 강의 03~10 내용을 실제로 연결한다

---

## 프로젝트 개요

### TODO 관리 프로그램

아주 단순한 TODO 관리 프로그램을 만든다.

기능 범위:

- TODO 추가
- TODO 조회
- TODO 삭제
- 테스트 코드 포함

기술 스택은 자유다.  
(Python, JavaScript 등 **본인이 익숙한 언어 사용 권장**)

---

## 디렉터리 구조 (권장)

```text
lab01_todo_basic/
├─ README.md
├─ src/
│  └─ todo.py        # 또는 todo.js
└─ tests/
   └─ test_todo.py
```

> 구조는 절대적인 정답이 아니다.  
> 중요한 것은 **Copilot에게 구조를 명확히 알려주는 것**이다.

---

## Step 1. Copilot에게 프로젝트 설명하기

Copilot Chat에 다음과 같이 입력해보자.

~~~text
이 lab는 학습용 TODO 프로젝트야.

목표:
- Copilot Coding Agent 실습
- 작은 기능을 단계적으로 구현

기능:
- TODO 추가
- TODO 조회
- TODO 삭제

테스트 코드도 함께 작성할 예정이야.
~~~

👉 이 단계는 **컨텍스트 주입**이다.

---

## Step 2. 기본 구조 생성 요청

이제 구조 생성을 맡긴다.

~~~text
위 설명을 기준으로
간단한 TODO 프로젝트의
초기 파일 구조와
각 파일의 역할을 제안해줘.
~~~

제안된 구조가 마음에 들면  
그대로 생성하도록 요청한다.

---

## Step 3. 첫 기능 맡기기 (추가 기능)

~~~text
TODO 항목을 추가하는 기능을 구현해줘.

조건:
- 간단한 데이터 구조 사용
- 테스트 코드 먼저 작성
- 변경 요약 제공
~~~

테스트가 실패해도 괜찮다.  
**실패가 기준을 만들어준다.**

---

## Step 4. 조회 기능 추가

~~~text
기존 TODO 목록을
조회할 수 있는 기능을 추가해줘.

조건:
- 기존 기능 유지
- 테스트 추가
~~~

👉 여기서 Copilot이  
기존 코드를 어떻게 이해했는지 관찰하자.

---

## Step 5. 삭제 기능 위임

~~~text
TODO 항목을 삭제하는 기능을 추가해줘.

조건:
- 존재하지 않는 ID 처리
- 테스트로 검증
~~~

이 단계는  
**09강(Build · Test · Ship)**과 직접 연결된다.

---

## Step 6. 리뷰 & QA 역할 전환

이제 역할을 바꿔본다.

~~~text
지금부터 Reviewer 역할로 전환해줘.
코드 품질과 구조를 리뷰해줘.
~~~

그리고,

~~~text
QA 역할로 전환해서
테스트 누락이나
엣지 케이스를 점검해줘.
~~~

---

## Step 7. 최종 정리

마지막으로 항상 이것을 요청하자.

~~~text
이번 lab에서 구현한 기능과
주의해야 할 점을
간단히 요약해줘.
~~~

이 요약은  
**미래의 나를 위한 문서**다.

---

## 이 Lab의 핵심 포인트

- 완벽한 코드가 목표가 아니다
- Copilot의 결과는 항상 다를 수 있다
- 중요한 것은 **대화 흐름과 판단 기준**이다
