# 07. Custom Instructions  
Copilot 행동 규칙 고정하기

이 강의에서는 Copilot Coding Agent를  
**‘그때그때 반응하는 AI’에서 ‘일관되게 행동하는 팀원’**으로 만든다.

핵심은 하나다.

> **Copilot의 기본 행동 규칙을 고정한다.**

---

## 이 강의의 목표

이 강의가 끝나면 다음을 할 수 있어야 한다.

- Copilot Custom Instructions의 개념을 이해한다
- 반복해서 말하던 지시를 기본 규칙으로 고정한다
- 개인/팀 단위 Copilot 사용 기준을 만든다

---

## 왜 Custom Instructions가 필요한가

Copilot을 계속 쓰다 보면 이런 일이 반복된다.

- “기존 구조는 건드리지 마”
- “변경 요약도 같이 줘”
- “테스트 없이 수정하지 마”

매번 말로 하기엔 피곤하다.  
그래서 **기본값으로 심어야 한다.**

> Custom Instructions는  
> Copilot의 ‘기본 성격’을 정의하는 설정이다.

---

## Step 1. Custom Instructions 위치 확인

VS Code에서 Copilot Chat을 열고,  
설정(Settings) 또는 Instructions 관련 메뉴를 찾는다.

환경에 따라 위치는 조금 다를 수 있지만,  
**“항상 적용되는 지시”**를 입력하는 곳이 있다.

---

## Step 2. 가장 단순한 규칙부터 넣기

처음부터 복잡하게 만들 필요는 없다.

다음처럼 아주 기본적인 규칙부터 시작한다.

~~~text
- 간결하게 답변할 것
- 코드 위주로 응답할 것
- 불필요한 설명은 생략할 것
~~~

이 설정만으로도  
Copilot의 말투와 출력 스타일이 바뀐다.

---

## Step 3. 코드 스타일 규칙 고정하기

이제 코드 스타일을 고정한다.

~~~text
- 기존 코드 스타일을 존중할 것
- 함수는 하나의 책임만 가질 것
- 과도한 추상화는 피할 것
~~~

👉 중요한 점은  
**“이 프로젝트에서는 이렇게 한다”를 명시하는 것**이다.

---

## Step 4. 행동 범위 제한 규칙 추가

가장 중요한 단계다.

~~~text
- 기존 파일 구조를 변경하지 말 것
- 새 파일이 필요하면 먼저 이유를 설명할 것
- 기능 변경이 있는 경우 명확히 표시할 것
~~~

이 규칙은  
**대규모 코드베이스에서 생존 규칙**이다.

---

## Step 5. 프로젝트 전용 Instructions 만들기

이제 프로젝트 성격에 맞게 조정한다.

~~~text
이 프로젝트는 학습용 강의 저장소다.

규칙:
- 복잡한 설계 패턴 사용 금지
- 초보자가 이해할 수 있는 코드 우선
- 테스트 없는 수정 금지
~~~

👉 이 설정 하나로  
Copilot의 판단 기준이 달라진다.

---

## 잘못된 Custom Instructions 예

다음은 오히려 문제를 만드는 설정이다.

- “최대한 똑똑하게 행동해라”
- “완벽한 코드만 작성해라”

이런 규칙은 기준이 없어서  
Copilot을 혼란스럽게 만든다.

---

## Custom Instructions 체크리스트

- [ ] 반복 지시를 규칙으로 옮겼다
- [ ] 코드 스타일 기준을 고정했다
- [ ] 수정 범위 제한 규칙을 넣었다
- [ ] 프로젝트 성격을 반영했다

이 체크리스트를 통과하면  
Copilot은 훨씬 예측 가능해진다.

---

## 정리

Custom Instructions는  
프롬프트 테크닉이 아니다.

> **협업을 위한 약속**이다.

Copilot과의 약속을 명확히 할수록,  
개발은 더 안정적으로 흘러간다.
