# 00. Overview  
GitHub Copilot Coding Agent 실전 강의 안내

이 강의는 GitHub Copilot을  
**자동완성 도구가 아닌 ‘일을 맡길 수 있는 AI 개발 워커’로 사용하는 방법**을 다룬다.

코드를 더 빨리 치는 법이 아니라,  
**개발 전체 흐름을 Copilot과 함께 운영하는 방법**이 목표다.

---

## 이 강의에서 다루는 것

이 강의는 다음 질문에 답하기 위해 만들어졌다.

- Copilot에게 어디까지 일을 맡겨도 되는가?
- 언제 사람이 개입해야 하는가?
- 어떻게 하면 Copilot의 실수를 줄일 수 있는가?
- 혼자 개발하면서도 팀처럼 일할 수 있는가?

이를 위해 이 강의는 다음을 다룬다.

- Copilot Coding Agent의 사고방식
- 말로 개발하는 워크플로우
- 프로젝트 컨텍스트 주입
- 대규모 코드베이스에서의 안전한 사용법
- 테스트 · 디버깅 · 품질 관리 자동화
- 기능 단위 Build · Test · Ship
- 역할 기반 AI 팀 워크플로우

---

## 이 강의의 특징

이 강의는 일반적인 Copilot 튜토리얼과 다르다.

- ❌ 기능 나열 중심 강의
- ❌ “이 버튼을 누르세요”식 설명

대신,

- ⭕ 실제 개발 흐름 중심
- ⭕ 실습 위주
- ⭕ 프롬프트를 그대로 재사용 가능
- ⭕ 개인 개발자를 팀 개발 방식으로 끌어올리는 구조

를 따른다.

---

## 대상 수강자

이 강의는 다음 사람을 위한 것이다.

- GitHub Copilot을 이미 사용해 본 개발자
- Copilot을 쓰지만 아직 확신이 없는 사람
- 혼자 개발하지만 팀 개발 방식이 필요한 사람
- AI로 개발 생산성을 구조적으로 높이고 싶은 실무자

프로그래밍 언어나 프레임워크에 대한 깊은 지식보다,  
**개발 경험과 문제 해결 경험**이 있는 사람이 가장 잘 맞는다.

---

## 강의 진행 방식

이 강의는 다음 구조로 진행된다.

1. 개념 최소 설명
2. 바로 실습
3. Copilot 결과 관찰
4. 잘못된 지시와 올바른 지시 비교
5. 재사용 가능한 패턴 정리

중요한 점은 다음이다.

> Copilot의 결과가  
> 강의 내용과 달라도 정상이다.

Copilot은 항상 같은 답을 내놓지 않는다.  
이 강의는 **정답을 재현하는 것**이 아니라  
**사고 방식과 사용 패턴을 익히는 것**이 목적이다.
