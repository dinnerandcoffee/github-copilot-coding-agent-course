# 04. Context Injection  
Copilot에게 프로젝트 맥락 주입하기

이 강의에서는 Copilot Coding Agent의 성능을  
**체감적으로 한 단계 끌어올리는 핵심 기술**,  
바로 **컨텍스트(Context) 주입**을 다룬다.

Copilot이 엉뚱한 답을 내놓을 때,  
대부분의 원인은 능력이 아니라 **맥락 부족**이다.

---

## 이 강의의 목표

이 강의가 끝나면 다음을 할 수 있어야 한다.

- Copilot에게 프로젝트 맥락을 설명한다
- README, 폴더 구조를 적극 활용한다
- Copilot의 오해를 교정한다
- “이 프로젝트를 이해한 Agent” 상태를 만든다

---

## 왜 컨텍스트가 중요한가

Copilot은 마법사가 아니다.  
현재 보고 있는 정보만으로 판단한다.

즉, 다음이 부족하면 결과도 흔들린다.

- 프로젝트 목적
- 전체 구조
- 사용 기술과 제약
- 변경하면 안 되는 영역

> **좋은 결과 = 좋은 맥락 + 명확한 지시**

---

## Step 1. 프로젝트를 요약하게 시켜라

가장 먼저 할 일은 이것이다.  
Copilot에게 **지금 이 프로젝트를 어떻게 이해하고 있는지** 말하게 한다.

~~~text
이 프로젝트의 목적과
전체 구조를 요약해줘.
각 폴더의 역할도 함께 설명해줘.
~~~

### 확인 포인트
- README를 참고하는가?
- 폴더별 역할을 구분하는가?
- 틀린 이해가 있는가?

👉 틀린 부분이 있다면 바로 잡아야 한다.

---

## Step 2. 잘못 이해한 맥락 교정하기

Copilot이 잘못 이해했다면,  
그냥 다시 시키지 말고 **정정 정보를 제공**한다.

~~~text
조금 정정할게.

- 이 프로젝트는 학습용 강의 저장소야
- 실습은 labs/ 디렉터리에서만 진행해
- course/는 문서이므로 코드 변경 대상이 아니야

이 정보를 반영해서 다시 이해를 정리해줘.
~~~

👉 이 과정이 **컨텍스트 주입**이다.

---

## Step 3. 하지 말아야 할 것을 명시하라

큰 프로젝트일수록  
“하지 말아야 할 것”이 중요하다.

~~~text
다음 제약을 반드시 지켜줘.

- course/ 디렉터리는 수정하지 말 것
- README 구조는 유지할 것
- 실습 코드는 labs/ 안에서만 변경
~~~

이 문장 몇 줄로  
Copilot의 행동 반경이 크게 줄어든다.

---

## Step 4. 컨텍스트를 전제로 작업 맡기기

이제 컨텍스트가 주입된 상태에서  
실제 작업을 맡겨본다.

~~~text
지금까지 이해한 프로젝트 맥락을 기준으로,

labs/ 디렉터리에
간단한 TODO 실습을 위한
초기 파일 구조를 제안해줘.
~~~

### 관찰 포인트
- 앞서 정리한 맥락을 활용하는가?
- 제약을 지키는가?

👉 이 순간부터 Copilot은  
**“이 프로젝트를 이해한 Agent”처럼 행동**한다.

---

## 흔한 실수

다음은 자주 발생하는 실수다.

- 컨텍스트 없이 바로 기능 요청
- Copilot의 오해를 방치
- README를 쓰지 않음

이 경우, 결과가 흔들리는 것은 정상이다.

---

## 컨텍스트 주입 체크리스트

- [ ] Copilot에게 프로젝트 요약을 시켰다
- [ ] 잘못된 이해를 교정했다
- [ ] 수정 금지 영역을 명시했다
- [ ] 맥락을 전제로 작업을 맡겼다

이 4가지를 거쳤다면  
컨텍스트 주입은 성공이다.

---

## 정리

컨텍스트 주입은  
프롬프트 테크닉이 아니다.

> **협업을 위한 기본 예의**다.

Copilot을 똑똑하게 만들려 하지 말고,  
**상황을 충분히 설명하자.**
